# 🤖 Infraestructura de Agentes (Kanarii)

> "Programar es humano, mantener la arquitectura es de agentes."

Esta carpeta contiene la **memoria colectiva** y las **instrucciones operativas** para los asistentes de IA (Agentes) que colaboran en el desarrollo de Kanarii.

## ❓ ¿Por qué existe esta carpeta?

En Kanarii usamos agentes de IA (como Gemini, Cursor, o Copilot) no solo para autocompletar código, sino para **garantizar la integridad de nuestra arquitectura DRY**. 

Si eres un colaborador humano y estás usando una IA para ayudarte, **DEBES** indicarle que lea el contenido de esta carpeta. Esto evitará que la IA proponga soluciones que rompan nuestra separación de capas o que use convenciones de nombres incorrectas.

## 📂 Estructura

- **`GEMINI.md`**: Reglas globales de comportamiento y principios core. Es la "constitución" de nuestros agentes.
- **`DEFINITION_OF_DONE.md`**: Lista de verificación que todo agente debe cumplir antes de entregar una tarea.
- **`rules/`**: Reglas específicas (Arquitectura, Nomenclatura, Git).
- **`skills/`**: Procedimientos paso a paso para tareas complejas (crear una feature, auditar seguridad, etc.).
- **`workflows/`**: Guías de flujo de trabajo para procesos repetitivos.

## 🤝 Contribución

Si descubres que un agente está cometiendo errores repetitivos o que una regla está desactualizada:
1. Mejora el archivo correspondiente en esta carpeta.
2. Haz un commit explicando la mejora.
3. ¡Felicidades! Has mejorado la inteligencia colectiva del proyecto.
